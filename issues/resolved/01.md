# End session in custom practice session does not work

## Original issue
The issue was reopened after browser logging showed only:

`[practice:end-session] end_session_button_pressed`

and none of the follow-up confirmation/store/service logs. That meant the button press was registered, but the confirmation step never invoked its callbacks in the browser.

## Resolution
- Confirmed the root cause by inspecting the local package implementation used by the web build:
  - `mobile/node_modules/react-native-web/dist/exports/Alert/index.js`
  - The web `Alert` implementation is a no-op (`static alert() {}`), so our `Alert.alert(...)` confirm dialog never rendered and its button callbacks never fired.
- Replaced the `Alert.alert(...)` end-session confirmation in `mobile/app/(tabs)/practice.tsx` with a cross-platform in-screen confirmation panel.
- Preserved the existing debug logs so the flow is still observable:
  - end-session button press
  - confirm panel open
  - cancel
  - confirm
  - async end-session start/finish
  - store/service summary state transitions
